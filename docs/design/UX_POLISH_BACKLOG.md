# UX Polish Backlog

## Purpose

This is a read-only UX polish audit of the complete staff appointment workflow. It reviews presentation, hierarchy, clarity, feedback, and interaction confidence. It does not propose workflow changes, new product features, or business-rule changes.

## Audit basis

- Walked the application as a staff member from landing through appointment entry, document selection, ID uploads, signatures, preview, draft, generation, and reset/navigation actions.
- Reviewed desktop at 1366x768 and 1920x1080, tablet at 768x1024, and mobile at 390x844.
- Reviewed blank, partially completed, test-data, optional-client, disabled-action, and long-page states.
- Severity reflects user hesitation and commercial-polish impact, not software defect priority.

## Smallest commercial-polish set

The following seven items provide the highest return without changing workflow:

1. **H1 — One readiness model:** align the visual hierarchy and vocabulary of header status, appointment summary, pending requirements, signatures, and checklist.
2. **H2 — Primary action hierarchy:** make Generate/Review/Download/Share read as a deliberate sequence and demote utilities.
3. **H3 — Footer obstruction:** preserve content visibility and dependable mobile touch targets around the fixed action bar.
4. **H5 — Section rhythm:** add consistent separation and heading cadence across the six long-form sections.
5. **H6/H7 — Readability and field width:** raise tiny operational type and ensure realistic values remain visually verifiable.
6. **H8 — Optional Client 2:** show a neutral Not required presentation until Client 2 exists.
7. **H10 — Persistent output/draft confidence:** show accurate saved/generated state beside existing actions after transient toasts disappear.

Together these changes would make the product feel substantially more deliberate, trustworthy, and commercially finished while retaining the current appointment workflow, controls, data flow, and business rules.

## Recommended sequencing

1. Align readiness/status vocabulary and optional Client 2 presentation.
2. Establish the action/button hierarchy and persistent generated/draft states.
3. Fix sticky-header/footer responsive density and obstruction.
4. Apply section spacing, type-scale, field-width, and heading consistency.
5. Polish preview, photo, signature, empty, checklist, toast, and alignment states.
6. Consider wide-desktop rebalancing only after the lower-risk system polish is validated.
