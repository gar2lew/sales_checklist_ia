# 2.7.0-alpha.1 Deployment Checklist

Production deployment is not authorised by completion of this checklist draft.

- [ ] Confirm branch `fix/staff-dropdown-seeding-v2` and approved RC commit.
- [ ] Confirm working tree is clean and contains no smoke artifacts.
- [ ] Confirm application version `2.7.0-alpha.1` and cache `v2.7.0-alpha.19`.
- [ ] Run all repository tests and record exact results.
- [ ] Confirm authoritative Phase 5 is 61/61 PASS.
- [ ] Complete and review the physical iPhone RC checklist.
- [ ] Resolve or formally accept every known issue.
- [ ] Obtain approval before pushing the branch.
- [ ] Create or update the pull request with test evidence.
- [ ] Complete code review and record the merge decision.
- [ ] Confirm the intended deployment target is not production until explicitly authorised.
- [ ] Identify and record the rollback commit/deployment.
- [ ] After authorised deployment, verify service-worker activation and old-cache cleanup.
- [ ] Complete production landing, workflow, document, download, share, email, and offline smoke tests.
- [ ] Create the approved release tag.
- [ ] Publish the approved release notes.
